1. PROJECT OVERVIEW
===========================================

This is a Django REST API backend for a KFC ordering system with integrated ElevenLabs voice AI capabilities.
The project enables customers to order food through voice calls (phone or browser-based) using conversational AI.

Key Technologies:
- Django 5.2.11
- Django REST Framework
- ElevenLabs Conversational AI SDK
- SQLite Database
- CORS enabled for frontend integration

3. DATABASE MODELS
===========================================

-------------------------------------------
3.1 Order Model (menu/models/Order.py)
-------------------------------------------

Fields:
- id: BigAutoField (Primary Key)
- customer_name: CharField(max_length=200)
- phone_number: CharField(max_length=50)
- address: TextField
- landmark: CharField(max_length=255, optional)
- items: JSONField (stores list of ordered items)
- total_price: IntegerField
- created_at: DateTimeField (auto_now_add=True)

Relationships:
- One-to-Many with Call model (reverse relation: order.calls.all())

String Representation:
"Order #{id} - {customer_name}"

-------------------------------------------
3.2 Call Model (menu/models/Call.py)
-------------------------------------------

Fields:
- id: BigAutoField (Primary Key)
- order: ForeignKey(Order, optional, on_delete=CASCADE)
- phone_number: CharField(max_length=50)
- call_type: CharField (choices: 'outbound', 'inbound', 'browser')
- conversation_id: CharField(max_length=255, unique=True)
- status: CharField (choices: 'initiated', 'ongoing', 'completed', 'failed')
- transcript: TextField (optional)
- duration_seconds: IntegerField (optional)
- metadata: JSONField (default={})
- created_at: DateTimeField (auto_now_add=True)
- updated_at: DateTimeField (auto_now=True)

Call Types:
- 'outbound': Outbound Phone Call (initiated by system)
- 'inbound': Inbound Phone Call (customer calls in)
- 'browser': Browser Voice Chat (web-based conversation)

Call Status Flow:
- 'initiated' → 'ongoing' → 'completed' OR 'failed'

Relationships:
- Many-to-One with Order (optional)

Meta:
- ordering: ['-created_at'] (newest first)

String Representation:
"Call {conversation_id} - {call_type} ({status})"

-------------------------------------------
5.2 ElevenLabsService Class
-------------------------------------------

Location: menu/services/elevenlabs_service.py

Methods:

1. __init__(self)
   - Initializes ElevenLabs client with API key
   - Sets agent_id from settings

2. initiate_phone_call(phone_number, context_data=None)
   - Initiates outbound phone call to customer
   - Uses: client.conversational_ai.create_agent_call()
   - Parameters:
     * phone_number: str (e.g., '+923001234567')
     * context_data: dict (optional metadata)
   - Returns:
     * success: bool
     * conversation_id: str
     * status: str
     * phone_number: str
     * error: str (if failed)

3. get_signed_token_for_chat(user_context=None)
   - Generates signed token for browser voice chat
   - Uses: client.conversational_ai.create_signed_url()
   - Parameters:
     * user_context: dict (optional metadata)
   - Returns:
     * success: bool
     * signed_url: str
     * agent_id: str
     * expires_at: str
     * error: str (if failed)

4. get_conversation_status(conversation_id)
   - Retrieves status of a conversation
   - Uses: client.conversational_ai.get_conversation()
   - Parameters:
     * conversation_id: str
   - Returns:
     * success: bool
     * conversation_id: str
     * status: str
     * transcript: str (optional)
     * duration_seconds: int (optional)
     * error: str (if failed)

-------------------------------------------
5.3 Integration Flow
-------------------------------------------

Outbound Phone Call Flow:
1. Frontend/User calls POST /calls/initiate/
2. Backend validates request (InitiateCallSerializer)
3. ElevenLabsService.initiate_phone_call() called
4. ElevenLabs API creates agent call
5. Call record created in database
6. conversation_id returned to frontend
7. ElevenLabs makes actual phone call
8. Webhooks update call status (POST /calls/webhook/)
9. Frontend polls GET /calls/status/<conversation_id>/

Browser Chat Flow:
1. Frontend calls POST /calls/chat-token/
2. Backend validates request (ChatTokenSerializer)
3. ElevenLabsService.get_signed_token_for_chat() called
4. ElevenLabs API generates signed URL
5. Call record created in database (browser type)
6. signed_url returned to frontend
7. Frontend uses ElevenLabs Web SDK with signed URL
8. User interacts with voice AI in browser
9. Webhooks update call status

13. HOW TO RUN THE PROJECT
===========================================

1. Setup Virtual Environment:
   python -m venv venv
   venv\Scripts\activate  # Windows
   source venv/bin/activate  # Linux/Mac

2. Install Dependencies:
   pip install -r requirements.txt

3. Setup Environment Variables:
   - Create/update .env file with:
     ELEVENLABS_API_KEY=your_key_here
     ELEVENLABS_AGENT_ID=your_agent_id_here

4. Run Migrations:
   python manage.py makemigrations
   python manage.py migrate

5. Create Superuser (optional):
   python manage.py createsuperuser

6. Run Development Server:
   python manage.py runserver

7. Access:
   - API: http://localhost:8000/
   - Admin: http://localhost:8000/admin/
   - Menu: http://localhost:8000/menu/
   - Orders: http://localhost:8000/orders/
